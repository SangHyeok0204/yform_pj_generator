지정된 영역의 코드를 리팩토링한다. $ARGUMENTS 에 대상 파일 경로 또는 영역을 전달한다.
만약 지정된 영역이 없을 경우, 전체 영역을 훑으며 리팩토링을 수행한다.


## 원칙
- 외부 동작(입출력, API, 사이드이펙트)은 절대 변경하지 않는다
- 지정된 영역만 수정한다. 관련 없는 코드는 건드리지 않는다
- CLAUDE.md의 Development Guidelines를 따른다

## 리팩토링 기준
- 중복 코드 제거
- 불필요한 복잡도 축소 (조건문 단순화, 중첩 감소)
- 함수/컴포넌트 분리 (단일 책임 원칙 위반 시)
- 네이밍 개선 (의도가 불명확한 변수/함수명)
- 타입 안전성 강화 (any 제거, 타입 좁히기)

## 절차
1. 대상 영역의 코드를 읽고 현재 구조를 파악한다
2. 리팩토링이 필요한 부분과 이유를 목록으로 정리하여 보여준다
3. 사용자 확인 후 리팩토링을 수행한다
4. 변경 사항을 요약한다
